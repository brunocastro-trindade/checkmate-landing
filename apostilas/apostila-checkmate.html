Visão Geral
do Projeto

Antes de escrever uma linha de código, precisamos entender o que vamos construir e como organizar os arquivos.

O que é uma Landing Page?

Uma landing page (página de destino) é uma página web criada com um objetivo claro: converter visitantes em leads ou clientes. Diferente de um site completo, ela é focada, direta e guia o usuário para uma única ação.

No caso da Checkmate Gestão, o objetivo é apresentar os serviços de gestão de tráfego pago e fazer o visitante entrar em contato ou solicitar uma proposta.

Estrutura de Arquivos

Vamos organizar o projeto com arquivos CSS separados por seção — isso facilita a manutenção e torna o código mais legível.

📁 checkmate-landing/
│
├── index.html ← página principal (toda estrutura HTML)
│
├── 📁 css/
│ ├── global.css ← reset, variáveis, tipografia base
│ ├── navbar.css ← estilos do header/navegação
│ ├── hero.css ← seção banner principal
│ ├── services.css ← cards de serviços
│ └── footer.css ← rodapé
│
└── 📁 assets/
├── 📁 images/ ← fotos e imagens
└── 📁 icons/ ← ícones SVG ou PNG
💡 Por que arquivos separados? Em projetos reais, separar o CSS por componente é uma boa prática. Fica fácil encontrar e editar o estilo de uma seção específica sem rolar centenas de linhas.

Como criar a estrutura

1

Crie a pasta raiz do projeto

No seu computador, crie uma pasta chamada checkmate-landing. Esta será a raiz do projeto.

2

Crie as subpastas

Dentro da pasta raiz, crie as pastas css e assets. Dentro de assets, crie images e icons.

3

Abra no VS Code

Abra o VS Code, vá em File → Open Folder e selecione a pasta checkmate-landing.

4

Crie os arquivos

No explorador lateral do VS Code, crie os arquivos: index.html, css/global.css, css/navbar.css, css/hero.css e css/services.css.

Boilerplate HTML

O esqueleto base do arquivo index.html com todas as ligações para os arquivos CSS externos.

O que é um boilerplate?

Boilerplate é o código inicial padrão que todo arquivo HTML precisa ter. Inclui a declaração do tipo de documento, configurações de idioma, charset, viewport e a importação dos arquivos CSS.

⌨️ Atalho no VS Code No arquivo index.html, digite ! e pressione Tab ou Enter. O VS Code gera automaticamente o boilerplate básico.

index.html — Estrutura Completa

index.html HTML 
<!-- Sempre começa com a declaração do tipo de documento -->
<!DOCTYPE html>
<html lang="pt-BR">

<head>
  <!-- Codificação de caracteres: suporta acentos e caracteres especiais -->
  <meta charset="UTF-8" />

  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

Tags Semânticas — Por que importam?

HTML5 trouxe tags que descrevem o significado do conteúdo, não apenas a aparência. Isso melhora o SEO (Google entende melhor sua página) e a acessibilidade.

TagSignificadoUso

<main>Conteúdo principalAgrupa todas as seções
<section>Seção temáticaHero, Serviços, Cases, etc.
<nav>NavegaçãoLista de links do menu
<footer>RodapéContato, copyright, links

Reset & Variáveis
CSS

O arquivo global.css é a base de todo o projeto — define as regras que se aplicam em todo lugar.

Por que precisamos de um Reset?

Cada navegador tem estilos padrão diferentes para elementos como margin, padding e tamanhos de fonte. O reset CSS garante que o site apareça consistente em todos os navegadores.

css/global.css CSS 
/* =============================================
   CHECKMATE GESTÃO — GLOBAL STYLES
   global.css | Base, Reset e Variáveis
   ============================================= */

/* --- Importação de Fontes --- */
@import url('https://fonts.googleapis.com/css2?family=Sora:wght@300;400;600;700&family=Inter:wght@300;400;500&display=swap');

/* --- Variáveis Globais (Design Tokens) --- */
:root {
  /* Cores */
  --color-primary: #C8102E;
  --color-primary-dark: #9e0d24;
  --color-black: #0D0D0D;
  --color-dark: #1A1A1A;
  --color-white: #FAFAFA;
  --color-gray: #F2F2F2;
  --color-gray-2: #E4E4E4;
  --color-text: #333333;
  --color-text-light: #777777;

  /* Tipografia */
  --font-title: 'Sora', sans-serif;
  --font-body: 'Inter', sans-serif;

  /* Tamanhos de fonte */
  --text-sm: 0.875rem;   /* 14px */
  --text-base: 1rem;      /* 16px */
  --text-lg: 1.25rem;    /* 20px */
  --text-xl: 1.5rem;     /* 24px */
  --text-2xl: 2rem;      /* 32px */
  --text-3xl: 2.5rem;    /* 40px */
  --text-4xl: 3.5rem;    /* 56px */

  /* Espaçamentos */
  --space-sm: 0.5rem;    /* 8px  */
  --space-md: 1rem;      /* 16px */
  --space-lg: 2rem;      /* 32px */
  --space-xl: 4rem;      /* 64px */
  --space-2xl: 6rem;     /* 96px */

  /* Bordas e Sombras */
  --radius: 8px;
  --radius-lg: 16px;
  --shadow: 0 4px 20px rgba(0, 0, 0, 0.08);
  --shadow-lg: 0 12px 40px rgba(0, 0, 0, 0.15);

  /* Largura máxima do container */
  --max-width: 1200px;
}

/* --- Reset Universal --- */
*,
*::before,
*::after {
  box-sizing: border-box;   /* padding e border incluídos no tamanho */
  margin: 0;
  padding: 0;
}

/* --- Rolagem suave ao clicar em links âncora --- */
html {
  scroll-behavior: smooth;
}

/* --- Estilos base do body --- */
body {
  font-family: var(--font-body);
  font-size: var(--text-base);
  color: var(--color-text);
  background-color: var(--color-white);
  line-height: 1.6;
  -webkit-font-smoothing: antialiased; /* deixa as fontes mais suaves */
}

/* --- Imagens responsivas por padrão --- */
img {
  max-width: 100%;
  display: block;
}

/* --- Links sem sublinhado --- */
a {
  text-decoration: none;
  color: inherit;
}

/* --- Utilitário: Container centralizado --- */
.container {
  width: 100%;
  max-width: var(--max-width);
  margin: 0 auto;          /* centraliza horizontalmente */
  padding: 0 var(--space-lg);
}
⚠️ Atenção O seletor * (asterisco) é um seletor universal — ele se aplica a TODOS os elementos da página. Usamos ele no reset para eliminar margens e paddings padrão do navegador.
