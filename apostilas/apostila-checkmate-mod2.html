Seção Sobre

Uma seção de dois painéis — imagem à esquerda e conteúdo de texto à direita — que apresenta a empresa, a filosofia e os diferenciais.

O que esta seção comunica

A seção "Sobre" serve para humanizar a empresa. Depois de ver os serviços, o visitante quer saber quem está por trás. Vamos usar um layout em duas colunas com foto e texto, além de uma lista de diferenciais com ícones.

📸
Coluna visual

Foto da equipe ou do espaço de trabalho. Gera confiança.

✍️
Headline + texto

Proposta de identidade. Quem somos, o que nos motiva.

✅
Diferenciais

Lista curta de vantagens competitivas com ícones.

🔢
Números

Dados de prova social: anos de experiência, projetos, etc.

Seção Contato

Formulário visual com envio direto para o WhatsApp via link — sem back-end necessário.

Envio para WhatsApp — como funciona?

O WhatsApp oferece um link especial no formato https://wa.me/55NUMERO?text=MENSAGEM. Podemos montar esse link dinamicamente com JavaScript, pegando os dados do formulário e codificando o texto com encodeURIComponent().

✅ Vantagem desta abordagem Não precisa de servidor, PHP, banco de dados ou serviço de e-mail. O visitante clica em "Enviar", o WhatsApp abre com a mensagem já preenchida. Simples, rápido e funcional para a fase inicial do negócio.

JavaScript — link para WhatsApp

Adicione este bloco <script> antes do </body> no seu index.html. Ele intercepta o envio do formulário e abre o WhatsApp com os dados preenchidos:

index.html — <script> antes de </body> JS 
<script>
  // Seleciona o formulário pelo id
  const form = document.getElementById('contactForm');

  // Número do WhatsApp da empresa (código do país + DDD + número)
  const WHATSAPP_NUMBER = '5547999999999'; // ← troque pelo número real

  // Escuta o evento de submit do formulário
  form.addEventListener('submit', function(event) {

    // Impede o comportamento padrão (recarregar a página)
    event.preventDefault();

    // Pega os valores dos campos
    const name    = document.getElementById('name').value;
    const whatsapp = document.getElementById('whatsapp').value;
    const service  = document.getElementById('service').value;
    const message  = document.getElementById('message').value;

    // Monta o texto da mensagem
    const text = `Olá! Meu nome é ${name}.
WhatsApp: ${whatsapp}
Interesse: ${service || 'Não informado'}
Mensagem: ${message || 'Sem mensagem adicional'}`;

    // Codifica o texto para funcionar em URL
    const encodedText = encodeURIComponent(text);

    // Abre o WhatsApp em nova aba
    const url = `https://wa.me/${WHATSAPP_NUMBER}?text=${encodedText}`;
    window.open(url, '_blank');
  });
</script>
🔍 Conceitos novos neste módulo addEventListener — escuta eventos (clique, submit, etc.) em um elemento.
preventDefault() — cancela o comportamento padrão do navegador.
encodeURIComponent() — converte texto em formato seguro para URL (espaços viram %20, etc.).
template literals — strings com `` ` `` que permitem inserir variáveis com ${variavel}.

Responsividade

Media queries CSS que adaptam o layout para tablets e celulares — sem quebrar o design visual.

O que são Media Queries?

Media queries são condições CSS que dizem: "aplique esses estilos SOMENTE quando a tela tiver tal tamanho". Com elas, o mesmo arquivo HTML funciona em qualquer dispositivo.

Breakpoints do projeto

Vamos usar três faixas de tela:

Breakpoints da Checkmate Landing Page
Desktop
≥ 1024px
Layout padrão. Todos os grids com múltiplas colunas.
Tablet
768px – 1023px
Grids de 4 colunas viram 2. Hero empilha vertical.
Mobile
≤ 767px
Tudo em coluna única. Navbar sem links visíveis.

Representação visual dos breakpoints

≥ 1024px Desktop
4 colunas
768px–1023px Tablet
2 colunas
≤ 767px Mobile
1 coluna

Sintaxe básica de media query

Sintaxe — como escrever CSS 
/* Estilos normais (sem media query) = desktop */
.services__grid {
  grid-template-columns: repeat(4, 1fr);
}

/* Media query para tablet: tela menor que 1024px */
@media (max-width: 1023px) {
  .services__grid {
    grid-template-columns: repeat(2, 1fr);  /* 4 → 2 colunas */
  }
}

/* Media query para mobile: tela menor que 768px */
@media (max-width: 767px) {
  .services__grid {
    grid-template-columns: 1fr;            /* 2 → 1 coluna */
  }
}
💡 Onde colocar as media queries? Existem duas abordagens: (1) adicionar as media queries no final de cada arquivo CSS separado, ou (2) criar um único arquivo css/responsive.css com todas elas. Para este projeto, vamos criar um arquivo separado para organização.

responsive.css — arquivo completo

Crie o arquivo css/responsive.css e adicione o link como último no <head> — ele deve sobrescrever os estilos anteriores.

Atualizar o <head> com os novos links

Aqui está como o bloco completo de links CSS deve ficar no seu index.html ao final deste módulo:

index.html — bloco <head> atualizado DIFF
<!-- CSS já existente (Módulo 1) --> <link rel="stylesheet" href="css/global.css" /> <link rel="stylesheet" href="css/navbar.css" /> <link rel="stylesheet" href="css/hero.css" /> <link rel="stylesheet" href="css/services.css" /> <link rel="stylesheet" href="css/about.css" /> <!-- novo --> <link rel="stylesheet" href="css/cases.css" /> <!-- novo --> <link rel="stylesheet" href="css/contact.css" /> <!-- novo --> <link rel="stylesheet" href="css/footer.css" /> <!-- novo --> <link rel="stylesheet" href="css/responsive.css" /> <!-- sempre por último! -->
⚠️ Ordem importa! O responsive.css sempre deve ser o último link. CSS é lido de cima para baixo — os estilos de baixo sobrescrevem os de cima quando há conflito. Se responsive.css vier antes, os outros arquivos vão anular as media queries.
