Animações CSS

Como usar @keyframes e animation para dar vida à página — sem uma linha de JavaScript.

Dois tipos de movimento no CSS

Antes de programar qualquer animação, é importante entender a diferença entre as duas ferramentas do CSS para movimento:

🔀
transition

Suaviza a mudança entre dois estados. Acionada por eventos como :hover ou adição de classe via JS. Já usamos nos botões e cards.

🎬
@keyframes + animation

Define uma sequência de estados com nome próprio. Roda automaticamente, pode repetir, pausar e ter delay. Ideal para entradas e efeitos contínuos.

Anatomia de uma animação CSS

Sintaxe — @keyframes e animation CSS 
/* PASSO 1: Definir os quadros da animação com @keyframes */
@keyframes fadeUp {
  from {                       /* estado inicial */
    opacity: 0;
    transform: translateY(30px);
  }
  to {                         /* estado final */
    opacity: 1;
    transform: translateY(0);
  }
}

/* PASSO 2: Aplicar a animação a um elemento */
.hero__title {
  animation-name:           fadeUp;  /* nome do @keyframes */
  animation-duration:       0.7s;    /* quanto tempo dura */
  animation-timing-function: ease-out; /* curva de velocidade */
  animation-delay:          0.1s;    /* esperar antes de iniciar */
  animation-fill-mode:      both;    /* manter estado final */

  /* Forma abreviada (shorthand): */
  animation: fadeUp 0.7s ease-out 0.1s both;
}
🎯 O que é animation-fill-mode: both? Sem esse valor, o elemento aparece no estado normal antes do delay e volta ao estado inicial depois de terminar. Com both, ele fica no estado inicial (from) durante o delay e fica no estado final (to) depois de terminar — que é o comportamento que queremos em 99% dos casos.

Scroll Reveal com
IntersectionObserver

A API nativa do navegador que detecta quando um elemento entra no campo de visão — sem bibliotecas externas.

O que é o IntersectionObserver?

É uma API JavaScript que observa elementos e dispara uma função toda vez que eles entram ou saem da área visível da janela (viewport). É a forma moderna e eficiente de fazer scroll reveal — muito mais leve do que calcular posições com window.scrollY a cada scroll.

👁️
observer

O objeto que fica "observando" os elementos registrados.

📦
entries

A lista de elementos sendo observados que mudaram de estado.

🎯
isIntersecting

Booleano: true quando o elemento está visível na tela.

🔖
threshold

Quanto do elemento precisa estar visível (0 = 1 pixel, 1 = 100%).

Código do Scroll Reveal

Este é o trecho JavaScript que observa todos os elementos com a classe .reveal e adiciona .visible quando eles aparecem na tela:

js/app.js — função scrollReveal() JS 
function scrollReveal() {

  // Configurações do observer
  const options = {
    root:       null,    // observa em relação à janela do navegador
    threshold:  0.12,   // dispara quando 12% do elemento estiver visível
    rootMargin: '0px'   // sem margem extra
  };

  // Cria o observer com uma função de callback
  const observer = new IntersectionObserver(function(entries) {

    // Para cada elemento observado que mudou de estado...
    entries.forEach(function(entry) {

      // Se o elemento está visível na tela:
      if (entry.isIntersecting) {

        // Adiciona a classe .visible (dispara a transição CSS)
        entry.target.classList.add('visible');

        // Para de observar — animação roda só uma vez
        observer.unobserve(entry.target);
      }
    });

  }, options);

  // Seleciona TODOS os elementos com a classe .reveal
  const elementos = document.querySelectorAll('.reveal');

  // Registra cada um no observer
  elementos.forEach(function(el) {
    observer.observe(el);
  });
}

// Chama a função quando a página termina de carregar
document.addEventListener('DOMContentLoaded', scrollReveal);
💡 Por que unobserve() depois de animar? Depois que o elemento ficou visível e a animação rodou, não precisamos mais observá-lo. O unobserve libera memória e deixa o código mais performático — especialmente em páginas com muitos elementos.

Navbar com
Efeito de Scroll

A navbar muda de aparência quando o usuário começa a rolar a página — um detalhe que transmite muita qualidade visual.

O efeito desejado

No topo da página, a navbar é totalmente preta (background: var(--color-black)). Quando o usuário rola mais de 80px, ela ganha uma borda sutil e um leve blur de fundo, criando uma sensação de profundidade — como se flutuasse sobre o conteúdo.

CSS — classe scrolled na navbar

Adicione este bloco no final do css/navbar.css:

css/navbar.css — adicionar ao final do arquivo CSS 
/* --- Estado da navbar após scroll --- */
.navbar.scrolled {
  background-color: rgba(13, 13, 13, 0.92);
  backdrop-filter: blur(12px);           /* efeito de vidro fosco */
  -webkit-backdrop-filter: blur(12px);   /* compatibilidade Safari */
  border-bottom-color: rgba(255, 255, 255, 0.1);
  box-shadow: 0 4px 24px rgba(0, 0, 0, 0.4);
}

/* Transição suave entre os dois estados */
.navbar {
  transition: background-color 0.3s ease,
               border-bottom-color 0.3s ease,
               box-shadow 0.3s ease;
}

JavaScript — detectar posição do scroll

Este trecho escuta o evento scroll da janela e adiciona ou remove a classe scrolled na navbar:

js/app.js — função navbarScroll() JS 
function navbarScroll() {
  const navbar = document.querySelector('.navbar');

  // Escuta o evento de scroll na janela inteira
  window.addEventListener('scroll', function() {

    // window.scrollY = quantos pixels o usuário rolou verticalmente
    if (window.scrollY > 80) {
      navbar.classList.add('scrolled');
    } else {
      navbar.classList.remove('scrolled');
    }
  });
}

document.addEventListener('DOMContentLoaded', navbarScroll);
🔍 Conceito: classList.add e classList.remove classList é a propriedade de todo elemento HTML que dá acesso às suas classes CSS. Os métodos add() e remove() adicionam ou removem classes dinamicamente — sem sobrescrever as outras classes que o elemento já tem. É a forma idiomática de usar CSS + JS juntos.

JavaScript — abrir e fechar o menu

js/app.js — função hamburgerMenu() JS 
function hamburgerMenu() {
  const btn  = document.getElementById('hamburger');
  const menu = document.getElementById('mobileMenu');

  // Clique no botão hamburguer
  btn.addEventListener('click', function() {
    const isOpen = btn.classList.contains('open');

    if (isOpen) {
      closeMenu();
    } else {
      openMenu();
    }
  });

  // Fechar ao clicar em qualquer link do menu mobile
  menu.querySelectorAll('a').forEach(function(link) {
    link.addEventListener('click', closeMenu);
  });

  // Fechar ao clicar fora do menu
  document.addEventListener('click', function(e) {
    const dentroDoMenu = menu.contains(e.target);
    const dentroDoBtn  = btn.contains(e.target);
    if (!dentroDoMenu && !dentroDoBtn) {
      closeMenu();
    }
  });

  function openMenu() {
    btn.classList.add('open');
    menu.classList.add('open');   // display: flex
    setTimeout(function() {
      menu.classList.add('visible'); // dispara transição de entrada
    }, 10);                         // pequeno delay para o display render
    btn.setAttribute('aria-label', 'Fechar menu');
  }

  function closeMenu() {
    btn.classList.remove('open');
    menu.classList.remove('visible');
    setTimeout(function() {
      menu.classList.remove('open'); // remove display após a transição
    }, 300);                          // aguarda 300ms (duração da transition)
    btn.setAttribute('aria-label', 'Abrir menu');
  }
}

document.addEventListener('DOMContentLoaded', hamburgerMenu);
💡 Por que o setTimeout de 10ms? O navegador precisa de pelo menos um ciclo de renderização para "ver" o elemento com display: flex antes de aplicar a transição CSS. Sem o timeout, a transição não funciona porque o navegador não viu o estado inicial. É um truque clássico para animar elementos que saem de display: none.

function hamburgerMenu() {

/* -------------------------------------------
   3. SCROLL REVEAL — IntersectionObserver
------------------------------------------- */
function scrollReveal() {
  const observer = new IntersectionObserver(function(entries) {
    entries.forEach(function(entry) {
      if (entry.isIntersecting) {
        entry.target.classList.add('visible');
        observer.unobserve(entry.target);
      }
    });
  }, { threshold: 0.12 });

  document.querySelectorAll('.reveal').forEach(function(el) {
    observer.observe(el);
  });
}

No repositório criado, clique em "uploading an existing file" e arraste toda a pasta do projeto. Escreva uma mensagem de commit como "Primeiro commit: landing page completa" e confirme.

3

Conectar Vercel ao GitHub

Acesse vercel.com, crie uma conta com o GitHub. Clique em "Add New Project", selecione o repositório checkmate-landing e clique em Deploy.

4

Configurar o Framework Preset

Quando aparecer a tela de configurações, selecione "Other" como framework (não é React, Next.js nem nada específico). Clique em Deploy e aguarde.

5

Site no ar!

Em cerca de 30 segundos, o Vercel gera uma URL no formato checkmate-landing-seu-usuario.vercel.app. Compartilhe com clientes e acesse de qualquer dispositivo.
